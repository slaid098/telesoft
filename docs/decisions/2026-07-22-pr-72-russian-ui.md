# ADR — PR 72: translate UI to Russian

## Статус

Accepted — 2026-07-22

## Контекст

telesoft — приложение для замены ссылок в постах Telegram-каналов.
Пользователи русскоязычные, но интерфейс был полностью на английском.
Новые компоненты (PreviewModal, PatternLibrary, ReplaceLinkForm) частично на
русском, старые страницы (login, layout, channels, jobs, ChannelForm) —
полностью на английском. ~95 user-facing строк в 14 файлах.

Решение о языке принято: русский как единственный язык интерфейса. i18n
framework (vue-i18n, svelte-i18n) не вводится — приложение одноязычное,
достаточно статичных строк.

## Решение

Перевести все user-facing строки на русский. Single source of truth для
job-status labels — `JOB_STATUS_LABELS` в `web/src/lib/types.ts` (используется
в badges на всех страницах). Остальные строки переведены inline в компонентах.

Не переведены (намеренно):
- `Telegram ID` — технический термин, русскоязычные пользователи используют
  "Telegram ID" / "телеграм-айди" в речи.
- `Username` — Telegram `@username` канала, оставлен как есть.
- URL placeholder-примеры (`https://t.me/bot?start=flow-*` и т.д.) —
  технические примеры.
- Бренд «telesoft».
- `ID` — идентификатор.

`app.html` `lang="en"` → `lang="ru"` для accessibility/SEO.

## Альтернативы

1. **Ввести i18n framework (svelte-i18n)** — overkill для одноязычного
   приложения. Усложнит код, добавит зависимость. Отвергнуто.

2. **Перевести только видимые страницы, оставить ошибки на английском** —
   error messages (Network error, Unauthorized) тоже видны пользователю.
   Отвергнуто — переведены все user-facing строки включая ошибки.

3. **Перевести Telegram ID → «Телеграм ID»** — теряет читаемость,
   "Telegram ID" привычнее. Отвергнуто — оставлен как есть.

4. **Перевести Username → «Имя пользователя»** — в контексте Telegram-канала
   `@username` это конкретный термин. Отвергнуто — оставлен как есть.

5. **Оставить Jobs → «Работы»** — буквальный перевод вводит в заблуждение
   (background jobs ≠ «работа»). Отвергнуто — переведено как «Задачи».
