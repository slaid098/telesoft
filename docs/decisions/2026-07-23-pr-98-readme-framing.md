# ADR PR#98 — README framing — link replacement as one feature

## Статус

Accepted

## Контекст

README описывал telesoft как софт ТОЛЬКО для замены ссылок («Редактор
постов Telegram-каналов — замена ссылок через бота-админа»). Проект
растёт: PR#96 добавил action menu + edit + is_active toggle на каналах —
управление каналами стало полноценным, не только список для замены ссылок.
В будущем планируются другие фичи. Старый framing блокировал расширение
и не соответствовал текущему состоянию.

## Решение

Framing README: telesoft — платформа для управления Telegram-каналами
через бота-админа. Замена ссылок описана как одна из фич (текущая, не
единственное назначение). Изменены ровно 3 места: tagline, первый абзац
секции «Описание», первые 2 буллета секции «Возможности» («Управление
каналами» первым, «Замена ссылок» как фича). Остальные секции не тронуты.

## Альтернативы

- **Оставить как есть** (status quo): README позиционирует софт как
  инструмент только для замены ссылок — не соответствует текущему
  состоянию (PR#96), блокирует расширение.
- **Отдельный docs-сайт** (mkdocs/Docusaurus): вынести framing на
  отдельный сайт, README оставить как есть. Избыточно для MVP, лишний
  maintenance overhead, нет аудитории для отдельного сайта.
- **Полный rewrite README**: переписать все секции под новый framing.
  Out of scope issue #97 — только framing, не переписывание. Остальные
  секции (архитектура, стек, деплой) актуальны как есть.
