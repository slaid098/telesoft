# ADR — PR 82: unified sticky top navigation on all devices

## Статус

Accepted — 2026-07-22

## Контекст

telesoft — приложение для замены ссылок в постах Telegram-каналов.
Навигация была разделена (PR #74): на десктопе — top nav в header
(`hidden sm:flex`), на мобиле — bottom tab bar (`grid grid-cols-2
... sm:hidden`, `aria-label="Мобильная навигация"`). Bottom tab bar
прикреплён к низу viewport'а через normal flow (последний элемент в
flex-column `<main>`).

Проблема: на страницах с длинным списком (например, 100 задач на
`/jobs`) bottom tab bar уезжает вниз вместе со скроллом — чтобы
переключиться на «Каналы», нужно прокрутить весь список до конца.
Навигация должна быть доступна мгновенно из любого места страницы.

Top nav на десктопе виден всегда (вверху страницы, не скроллится
вместе с контентом, т.к. находится вне `overflow-y-auto` контейнера),
но скрыт на мобиле (`hidden sm:flex`) — там用户 видел только bottom
tab bar.

Issue #81 просит объединить: единая навигация в шапке на всех
устройствах, шапка sticky (приклеена к верху, двигается при скролле).

## Решение

Объединить навигацию в единую sticky шапку на всех устройствах,
нижний таб-бар удалить:

1. **Top nav всегда видим** — с `<nav>` убрано `hidden sm:flex` →
   `flex items-center gap-2`. Nav из двух пунктов («Каналы»,
   «Задачи») помещается в шапку на мобиле рядом с wordmark и кнопкой
   «Выйти». Tap targets адаптированы для touch: `py-2` → `py-2.5`
   (крупнее область нажатия, WCAG 2.5.5). `space-x-1` → `gap-2`
   (flex+gap единообразно с остальной шапкой).

2. **Header `sticky top-0 z-50`** — header приклеен к верху viewport'а,
   двигается вместе со скроллом страницы, всегда виден. `sticky` (не
   `fixed`) — header остаётся в normal flow, отдельный `padding-top`
   для контента не нужен. `z-50` поднимает header над контентом при
   скролле. Background остался непрозрачным `bg-slate-900` (как и был)
   — контент не просвечивает.

3. **Удалён bottom tab bar** — блок `<nav class="grid grid-cols-2
   ... sm:hidden" aria-label="Мобильная навигация">` удалён полностью.
   `aria-label` удалён вместе с блоком (больше нигде не используется).
   Дублирование разметки (desktop top nav + mobile bottom tab bar)
   устранено — одна разметка навигации для всех устройств.

4. **Layout адаптирован** — контент `<main>` (`overflow-y-auto` +
   `flex-1` в flex-column контейнере) корректно скроллится под sticky
   header. Отдельный `padding-bottom` для bottom bar отсутствовал
   (контент обрезался `overflow-y-auto`), убирать нечего.

5. **Тест `layout.test.ts` не обновлялся** — assertions на «Каналы»
   (`getAllByText`, `>=1`) и «Выйти» (`getByRole button`) проходят;
   assertions на `Мобильная навигация`, `grid-cols-2` или bottom tab
   bar отсутствовали.

## Альтернативы

1. **Оставить split (desktop top nav + mobile bottom tab bar)** —
   текущее состояние (PR #74). Bottom tab bar неудобен на длинных
   списках: переключение требует скролла в самый низ. Issue #81 явно
   просит убрать split. Отвергнуто.

2. **Hamburger menu на мобиле** — иконка-бургер в шапке, по тапу
   раскрывается dropdown с пунктами навигации. Решает проблему
   видимости (бургер всегда в шапке), но добавляет лишний тап для
   переключения (открыть меню → выбрать пункт) и скрывает навигацию
   за дополнительным жестом. Для двух пунктов («Каналы», «Задачи»)
   hamburger избыточен — пункты помещаются в шапку напрямую. Отвергнуто
   (over-engineering для текущего количества пунктов; может быть
   пересмотрено если пунктов станет 5+).

3. **Fixed bottom bar (всегда виден, не скроллится)** — `fixed bottom-0`
   на bottom tab bar, чтобы он не уезжал вниз со скроллом. Решает
   проблему видимости, но: занимает место внизу экрана постоянно,
   закрывает контент на узких экранах, конфликтует с браузерными
   панелями на iOS Safari (bottom safe area). Sticky top header
   предпочтительнее по UX-конвенции (навигация вверху — стандарт для
   веб-приложений, не нативных мобильных). Отвергнуто.

4. **Floating action button / quick-switch** — FAB для быстрого
   перехода между разделами. Не решает проблему видимости текущего
   раздела, добавляет нестандартный UI-элемент. Отвергнуто.
