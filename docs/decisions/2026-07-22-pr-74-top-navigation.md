# ADR — PR 74: replace sidebar with top navigation bar

## Статус

Accepted — 2026-07-22

## Контекст

telesoft — приложение для замены ссылок в постах Telegram-каналов.
Навигация была реализована как левый сайдбар (`<aside>` шириной `w-60`)
с двумя пунктами (Каналы, Задачи) на desktop, плюс нижний таб-бар
(`grid-cols-2`) на mobile. Сайдбал занимал место по ширине, дублировал
навигацию с mobile tab bar, и не содержал ссылки «Главная» (домой).
Пользователь захотел верхнюю навигацию: Главная, Каналы, Задачи — в
самом верху страницы (issue #73).

`/` редиректит на `/channels` (через `web/src/routes/+page.ts`), поэтому
«Главная» и «Каналы» ведут на одну страницу. Это ок — «Главная» нужна
семантически как «домой» и для быстрого возврата к корню.

## Решение

Заменить сайдбар верхней навигационной панелью в `web/src/routes/+layout.svelte`:

1. **Убрать `<aside>`** — layout становится одноколоночным
   (`flex flex-col`, без `sm:flex-row`).

2. **Превратить `<header>` в верхний навбар** — слева wordmark «telesoft»
   (кликабельный `<a href="/">`), горизонтальный `<nav>` (Каналы, Задачи)
   на desktop `sm+`, справа «Вы вошли как {username}» + кнопка «Выйти»
   (перенесена из сайдбара).

3. **Активное состояние** — хелпер `isActive(item)`:
   - для `/` (Главная) — строгое `pathname === "/"` (иначе `startsWith("/")`
     сработает на всех страницах);
   - для остальных — `pathname.startsWith(item.href)` (как было).
   Хелпер используется и в desktop nav, и в mobile tab bar.

4. **Mobile таб-бар** — `grid-cols-2` → `grid-cols-3` (три пункта: Главная,
   Каналы, Задачи), активное состояние через тот же `isActive`.

5. **Login guard** `{#if isLogin}` сохранён — login рендерится без оболочки.

## Альтернативы

1. **Оставить сайдбар, добавить «Главная» в него** — сайдбал занимал место
   по ширине и дублировал навигацию с mobile tab bar. Верхняя навигация
   унифицирует desktop/mobile и освобождает ширину. Отвергнуто.

2. **Tabs-only (убрать desktop навигацию, оставить только mobile tab bar)** —
   desktop-пользователи теряют постоянную навигацию (tab bar скрыт на `sm+`).
   Отвергнуто — верхний навбар виден на всех размерах экрана.

3. **Hamburger menu (бургер-меню)** — прячет навигацию за иконкой, требует
   лишний клик для доступа. Для 3 пунктов избыточно. Отвергнуто —
   горизонтальный nav в header показывает все пункты сразу.

4. **Не добавлять «Главную», оставить 2 пункта (Каналы, Задачи)** — нет
   способа быстро вернуться «домой» без правки URL. Issue #73 явно просит
   «Главная». Отвергнуто — «Главная» добавлена.
