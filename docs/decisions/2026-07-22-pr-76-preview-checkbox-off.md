# ADR — PR 76: preview checkbox off by default

## Статус

Accepted — 2026-07-22

## Контекст

telesoft — приложение для замены ссылок в постах Telegram-каналов.
Форма замены (`web/src/lib/components/ReplaceLinkForm.svelte`) имела две
кнопки рядом: «Предпросмотр» и «Запустить». Пользователь сказал, что
предпросмотр «портит весь вид» — кнопка предпросмотра всегда была видна
рядом с основной кнопкой «Запустить». Нужно сделать предпросмотр
опциональным: по умолчанию выключен, включается пользователем (issue #75).

## Решение

Добавить чекбокс «Показать предпросмотр» и обернуть кнопку «Предпросмотр»
в условный рендер `{#if showPreview}`:

1. **State `showPreview`** — `let showPreview = $state(false)`, по
   умолчанию выкл.

2. **Чекбокс** — `<label class="flex items-center gap-2 text-sm
   text-slate-300"><input type="checkbox" bind:checked={showPreview} />
   Показать предпросмотр</label>`, размещён в строке кнопок
   (`flex items-center justify-end gap-2`) перед кнопками. Ненавязчивый,
   использует существующие Tailwind-классы (как у radio-кнопок режима
   замены выше в той же форме).

3. **Условная видимость кнопки «Предпросмотр»** — обёрнута в
   `{#if showPreview}`, рендерится только при включённом чекбоксе.

4. **Кнопка «Запустить» всегда видна** — вне `{#if}`, не зависит от
   `showPreview`.

## Альтернативы

1. **Всегда показывать preview-кнопку (как было)** — пользователь явно
   сказал, что предпросмотр «портит весь вид». Отвергнуто.

2. **Показывать preview-кнопку по hover на форме** — неявное поведение,
   пользователь может не догадаться навести курсор. Трудно обнаружить.
   Отвергнуто — чекбокс явный и управляемый пользователем.

3. **Modal toggle (скрывать предпросмотр в модалку, открываемую кнопкой
   в другом месте)** — требует отдельной кнопки-триггера и модального
   диалога, что усложняет UI и расходится с задачей (сделать предпросмотр
   опциональным в текущем месте, а не перемещать). Отвергнуто.
