# ADR — PR <PR-NUMBER>: remove duplicate Home link from top navigation

## Статус

Accepted — 2026-07-22

## Контекст

telesoft — приложение для замены ссылок в постах Telegram-каналов.
Верхняя навигация (PR #74) состояла из трёх ссылок: wordmark
«telesoft» (`href="/"`), пункт «Главная» (`href="/"`) и «Каналы»
(`href="/channels"`). Корневой путь `/` редиректит на `/channels`
(через `web/src/routes/+page.ts`), поэтому все три ссылки вели на одну
и ту же страницу. Это дублирование сбивало с толку — три визуально
разных элемента делали одно и то же (issue #79).

Wordmark «telesoft» (кликабельный логотип) уже выполняет роль «домой»
по UX-конвенции. Отдельный пункт «Главная» был избыточен и дублировал
wordmark и «Каналы».

## Решение

Убрать пункт «Главная» из навигации, wordmark оставить как «домой» и
направить сразу на `/channels` (минуя редирект):

1. **Убрать «Главная» из `navItems`** — в `web/src/routes/+layout.svelte`
   удалён `{ href: "/", label: "Главная", icon: "🏠" }`. Остались
   «Каналы» и «Задачи».

2. **Wordmark → `/channels`** — `<a href="/">` изменён на
   `<a href="/channels">`. Wordmark выполняет роль «домой» (стандартный
   UX-паттерн: кликабельный логотип ведёт на главную). Теперь ведёт
   сразу на `/channels`, без промежуточного редиректа `/` → `/channels`.

3. **Упростить `isActive`** — строгое сравнение `pathname === "/"`
   для «Главная» больше не нужно (нет пункта с `href: "/"`). Хелпер
   упрощён до `pathname.startsWith(item.href)` для всех оставшихся
   пунктов. Хелпер оставлен (не inline) — единая точка логики для
   desktop nav и mobile tab bar.

4. **Mobile таб-бар `grid-cols-3` → `grid-cols-2`** — два пункта вместо
   трёх, сетка адаптирована под новое количество.

5. **Тест `layout.test.ts` не обновлялся** — assertions на «Каналы» и
   «Выйти» проходят; assertions на «Главная» или `grid-cols-3`
   отсутствовали.

## Альтернативы

1. **Убрать wordmark (оставить «Главная» и «Каналы»)** — wordmark
   «telesoft» несёт брендинг и является стандартным местом для «домой»
   по UX-конвенции. Убирать его — потерять визуальную идентификацию
   приложения. Отвергнуто.

2. **Убрать «Каналы» (оставить wordmark и «Главная»)** — «Каналы» —
   основная рабочая страница приложения (список каналов), её пункт
   навигации должен быть явным. «Главная» (`/`) семантически
   дублировала бы wordmark. Отвергнуто.

3. **Оставить дублирование (wordmark + «Главная» + «Каналы»)** — три
   ссылки ведут на одну страницу, это сбивает с толку и захламляет
   навигацию. Issue #79 явно просит убрать дублирование. Отвергнуто.
