## Patterns

- **src layout** (`src/telesoft/` вместо `telesoft/`) — изолирует пакет от корня репо
- **Frozen dataclass** для конфига — иммутабельный `Settings`, `from_env()` читает переменные окружения
- **Lifespan** для инициализации/закрытия ресурсов — `init_db()` + `EventBus`/`JobRunner` start + `start_client()` (try/except) на startup, `runner.stop()` + `stop_client()` (try/except) + `close_db()` на shutdown. Падение Telegram НЕ роняет app (логируется через loguru)
- **Health endpoint** `GET /health` — стандартный liveness probe (public, без auth)
- **Env-префиксы**: `ADMIN_`, `SECRET_KEY`, `TELEGRAM_`, `JOBS_MAX_CONCURRENCY`
- **Raw aiosqlite без ORM** (паттерн media-gen) — `CREATE TABLE IF NOT EXISTS` на startup, без миграций
- **`type Row = dict[str, Any]` (PEP 695)** — ruff UP040 запрещает `TypeAlias`
- **`@dataclass _State` holder** вместо `global` (ruff PLW0603) — singleton state в `db.connection._state.db` и `core.telegram._state.client/started`
- **`_CREATE_SQL` + `_CREATE_INDEXES_SQL`** константы на модуль — `init_db()` итерирует в порядке FK-зависимостей (channels → edit_jobs → edit_logs)
- **FK CASCADE** с `PRAGMA foreign_keys=ON` (default OFF в SQLite) — канал → job'ы → логи
- **WAL mode** для конкурентного чтения при write-операциях
- **`update_channel` whitelist** (`title`/`username`/`is_active`) — `ValueError` на неизвестном поле
- **CRUD функции принимают `aiosqlite.Connection` первым аргументом** — no global state в моделях
- **Bot-mode Telethon singleton** (`core/telegram.py`) — `TelegramClient(session, api_id, api_hash, receive_updates=False)` + `client.start(bot_token=...)`. Token в `start()`, НЕ в конструктор. **PR#48**: `session = StringSession(settings.telegram_session_string) if settings.telegram_session_string else StringSession()` — `TELEGRAM_SESSION_STRING` env var переиспользует auth_key между restarts (prevents FloodWait, supersedes PR#46 in-memory `StringSession()` без saved auth). Empty string → fallback to `StringSession()` (local dev / first run). Поле `Settings.session_path` сохранено для backwards compat, но не используется (deprecated PR#46)
- **By-ID fetch only** (для существующих методов) — `get_messages(chat_id, ids=[...])`. **КРИТИЧНО: НЕ `iter_messages`/`get_messages(limit=...)`** — `BotMethodInvalidError` для ботов (history iteration запрещён MTProto)
- **`get_last_messages` user-provided max_id** (PR#32 auto-discovery, PR#34 perf, **PR#62 заменил binary search на user-provided post_link**) — `get_last_messages(channel_id, limit=100, max_id=0)` через raw API `channels.GetMessagesRequest` (namespace `telethon.tl.functions.channels`, alias `ChannelsGetMessagesRequest`). Работает для bot-admin (в отличие от `iter_messages`). **PR#62**: `max_id` передаётся caller'ом (из `parse_post_link(payload.post_link)`), `ids = list(range(max_id, max(0, max_id - limit), -1))`, один fetch через `_fetch_messages_by_ids`. `max_id=0` → `[]` с warning (caller обязан предоставить валидный post_link). Binary search `_find_max_id` удалён (ломался на дырах удалённых постов — посты 114-138 удалены, 139-140 существуют, binary search возвращал 113). `Settings.max_probe_id` / env `MAX_PROBE_ID` удалены. Снимает ограничение PR#14 (by-ID fetch only) — fetch последних N постов без user-session, но требует user input (post_link)
- **`_get_channel_input(channel_id) -> InputChannel`** — `get_entity(channel_id)` → `InputChannel(entity.id, getattr(entity, "access_hash", 0))` (PR#34: defensive `getattr` default 0 — если `get_entity` вернёт не-Channel, `access_hash` может не быть → 0 вместо AttributeError). Raw API требует `InputChannel` (id без `-100` prefix + access_hash)
- **`_fetch_messages_by_ids` filter** — `m is not None and getattr(m, "date", None) is not None` фильтрует `None` (пост удалён/не существует) и `Message` с `date=None` (служебные). Возвращает `ChannelMessages.messages` (filtered)
- **FloodWaitError retry max 3** — try/except в `_fetch_messages_by_ids`, `await asyncio.sleep(exc.seconds + 1)` между retry, после 3 попыток re-raise (bounded retries, НЕ infinite как в spike PR#30). Caller решает retry/log/abort
- **`parse_post_link(link) -> int`** (PR#62) — извлекает message_id из Telegram post link или plain number. `re.search(r"/(\d+)/?$", link.strip())` → если match, `int(match.group(1))` (matches `https://t.me/channel/140`, `https://t.me/c/1234567890/140`, trailing `/N`); иначе `int(link.strip())` fallback (plain number `140`); иначе `ValueError(f"cannot parse post link: {link}")` → router 422. URL с query `?comment=...` — regex `/?$` anchor matches `/N` before query
- **`asyncio.sleep(settings.telegram_request_delay)`** между запросами к Telegram (default 1.0s) — после каждого probe в `_find_max_id` (КРОМЕ последнего, PR#34) и перед range fetch в `get_last_messages`. **PR#62**: `_find_max_id` удалён, delay используется только в `_fetch_messages_by_ids` FloodWait retry (`exc.seconds + 1`). НЕ уменьшать delay (flood risk)
- **`asyncio.Lock` + double-checked locking** в `start_client()` — защита от concurrent start (fast path до lock, slow path повторная проверка внутри `async with _connection_lock`)
- **`edit_message` propagates `RPCError`** — не глотает ошибки, caller решает retry/log/abort. **PR#64**: `edit_message(chat_id, message_id, text, formatting_entities=None)` — опциональный `formatting_entities` kwarg передаётся в `client.edit_message(formatting_entities=...)` для сохранения bold/italic/link entities при text-замене (см. `_adjust_entity_offsets` в link_replacer.py)
- **`edit_message_entities` raw API** (PR#44) — `messages.EditMessageRequest` (из `telethon.tl.functions.messages`, НЕ `channels` — `channels.EditMessageRequest` не существует в Telethon 1.44) с `peer=InputPeerChannel(channel_id, access_hash)`. Channel input resolved через `_get_channel_input` (InputChannel → InputPeerChannel conversion). Используется когда URL в `MessageEntityTextUrl` entity (formatted link) — `client.edit_message` обновляет только text, не entities. Возвращает raw `Updates` результат

Для каждого match of pattern in text: match strictly before entity → shift `entity.offset` by cumulative delta (`len(new_link) - len(match)`); match strictly inside entity → adjust `entity.length` by per-match delta. **PR#66**: crossing boundary теперь обрабатывается — case 3a (match starts outside entity, ends inside) → `new_offset = m_start + len(new_link)`, `e_length -= new_offset - e_offset`, `e_offset = new_offset` (entity начинается после replacement); case 3b (match starts inside entity, ends outside) → `e_length += len(new_link) - (m_end - m_start)` (entity extended/shrunk чтобы покрыть new link). Defensive validation post-adjustment: entities с `offset < 0` OR `length <= 0` OR `offset + length > len(new_text)` → dropped (не appended), `new_text = re.sub(pattern, new_link, text)` вычисляется внутри функции для validation.
